# 受講生管理スプレッドシート — 設計仕様

**目的**: 運営・メンターが **1 ブック**で「進捗」と「出欠」を追う。合同イベントの営業用マスタ（`operations/合同イベント_商品群_マスタ.csv`）と**イベント名の表記**を揃える。

## 共有・自動化

- **サービスアカウント**（読み取り／自動化用）: `cursor-sheets-service@august-now-482801-j4.iam.gserviceaccount.com` — **スプレッドシートを編集者で共有**。JSON キーはリポジトリに含めない。
- **参照テンプレ**（体裁のお手本）: 運営が保持する既存シートに合わせて列名を微調整してよい。  
- **スケジュール用テンプレ（イメージの正）**: [Google スプレッドシート（参考）](https://docs.google.com/spreadsheets/d/1RmsxI6HEceUKHBALR-Otv4t9jNVKmPHETHjDmuBtEUc/edit?gid=928801667#gid=928801667) — 本コースは **2週間版**として **同じノリで** `カレンダー_第N期` 等を新規ブックに複製・圧縮して作る（インタビュー反映・[`EVENT_CONTENT_INTERVIEW.md`](../docs/EVENT_CONTENT_INTERVIEW.md) の **5/9 までに埋め尽くし**と整合）。

## 推奨タブ構成

| タブ名 | 用途 |
|--------|------|
| `README` | シートの読み方・個人情報の取り扱い・更新ルール |
| `ロースター` | 受講生 1 人 1 行のマスタ（氏名・連絡・班・進捗セグメント） |
| `進捗セグメント定義` | ドロップダウン用の一覧と意味 |
| `カレンダー_第N期` | 2 週間のキー日程（Day0〜Day14・火木 OH・Day13 まくり） |
| `スケジュール_2週間_受講生`（名称は運営で統一） | **受講生 1 人が「いつ何をするか」**のチェック用。**達成率**の定義は**このシートに行／列が並んでから**決める（[`EVENT_CONTENT_INTERVIEW.md`](../docs/EVENT_CONTENT_INTERVIEW.md)） |
| `出欠_イベント` | 列: 受講生ID / Day0 / スポット / Day1 / Day7 / Day13 / Day14 …（○・×・オンライン・オフライン） |
| `出欠_勉強会OH` | 列: 受講生ID / 各日付の各回（火木×最大 4 回） |
| `テンプレ_MVP1枚` | コピー用の見本行（MVP 定義・公開 URL・リポジトリ） |
| `テンプレ_友達FB` | コピー用（誰に / フィードバック / 反映したこと） |

## 進捗セグメント（例・データ入力規則）

| 値 | 意味 |
|----|------|
| S0 未着手 | 環境未セットアップ |
| S1 環境OK | Cursor / GitHub / ローカル実行まで到達 |
| S2 実装中 | コア機能開発中 |
| S3 デプロイ中 | URL 公開に向けた調整 |
| S4 MVP完了 | チェックリスト必達を満たす |
| S5 最終提出済 | Day14 提出・デモ完了 |

## イベント名の揺れ防止（例）

- `YYYY-MM-DD_OH_火` / `YYYY-MM-DD_OH_木`
- `Day0` `Spot` `Day1` `Day7` `Day13` `Day14`
