**目的**: **同一フォーム**で第1期・第2期を扱い、**申込順（先着）**でコホートを振り分ける。**満員後の新規申込は第2期への誘導のみ**。**Day0 は完全オンライン**で、**Day0〜Day1 の間**に GitHub / Supabase / Cursor Pro 等の準備を**社員またはメンターが個別フォロー**する（[`SETUP_CHECKLIST.md`](SETUP_CHECKLIST.md)）。

## 1c. 確認メールの目的（運営の共通理解）

受講生向け **確認メール**は、少なくとも次を満たす。

1. **申込が受理されたこと**の証明（いつ・何に申し込んだか）。  
2. **第1期／第2期のどちらに入ったか**の明示。  
3. **次の一手**（Day0 の日時・連絡手段・[`SETUP_CHECKLIST.md`](SETUP_CHECKLIST.md) や FAQ への導線）。  
4. **問い合わせ先**。

**文言は主催が最終確認**する。

## 1d. 満員時・自動返信の方針（ドラフト）

- **本文は主催が書く**。  
- **FAQ へのリンク**を必ず含める（詳細は FAQ 正）。  
- **日程が一目でわかる**よう、**「日程はこの表です」**のように **表または短い要約**を同梱する（第2期の日付が伝わること）。  
- **キャンセル待ち**は案内しない。

6. 端末トラブル時は **運営に連絡**（**PC レンタル**は**予算検討時に運営で判断**・[`POLICY_SCALE_AND_COSTS.md`](POLICY_SCALE_AND_COSTS.md)）  
7. 次のアクション（OH・連絡手段）
