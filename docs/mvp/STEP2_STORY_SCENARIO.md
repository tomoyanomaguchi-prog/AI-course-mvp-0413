# Step2 — 利用ストーリー・画面で確認できること・制約（メンター提出用）

**目的**（[`STEPS_MVP_FLOW.md`](STEPS_MVP_FLOW.md)）: Step1 の内容を材料に、**誰が何をするか**・**各段階で画面に何が見えればよいか**という **ストーリー** を先に押さえる。同じ工程で **技術制約**（標準スタック・緑/黄）と **開発工数の見立て** を反映し、**難しすぎる機能を除外**する。

**この段階でやらないこと**: 本番の Next 実装、`app/` への追加、本物のログイン／DB 接続。**Step3** で外側モックを作る。

---

## 1. ストーリー一行

**28卒の就活生が、新しい応募先を一覧に追加し（または既存を選び）、企業名に応じて想定されるESの設問を画面に出し（LLM・推測）、材料を貼ったうえで叩きを生成し、編集して保存し、一覧に戻って別の企業でも繰り返す。**（設問生成を後回しにする場合は、材料→叩きの最短線だけ先に通す。）

## 3. トリガー → ステップ → ゴール（主導線）

| # | ユーザーがすること | この時点で「うまくいった」と言えること |
|---|---------------------|----------------------------------------|
| 1 | **応募先企業を新規追加する**／**一覧から既存の企業行を選ぶ** | 一覧に自分が登録した企業名が見え、開ける |
| 2 | **企業名を確定したあと**、**想定ES設問が並ぶ（LLM・推測）** → **材料（過去のES・メモ）を貼る／設問欄に入力** → **叩きを生成（LLM）** → 必要なら手で直す | 設問が**参考用として**表示され、生成結果が画面に出て編集できる |
| 3 | **（ゴール）材料・設問への回答・叩きを保存し、一覧に戻る** | 保存後、一覧から同じ企業を開くと**前回の内容が残っている** |

## 4. 画面ごとに「何が確認できるか」（ストーリーと対応）

Step3 のモック・Day7 の G4 のたたきになる。**最低2画面**。各画面について「見せる情報」と「確認できること」だけ書く（デザイン詳細は Step3）。

| 画面ID | 画面の役割（名前） | ユーザーが確認できること（箇条書き） |
|--------|---------------------|----------------------------------------|
| **S1** | **応募企業一覧** | ・登録済み企業名のリスト（行）<br>・**企業を追加**するための入力と操作<br>・一覧の1件を選ぶと **S2** に進めること<br>・（任意）更新日時が分かること |
| **S2** | **企業ワークスペース（想定設問・材料・叩き）** | ・どの企業向けか（企業名が分かる）<br>・**想定ES設問**のリスト（動的。**実際の募集要項と異なる場合がある**旨の注記）<br>・**材料**用の複数行テキスト（設問への回答を材料に含めてもよい）<br>・**叩きを生成**する操作と、**生成結果（編集可能）**<br>・**保存**で永続化されること、**一覧へ戻る**導線 |

（**S3 は置かない**。設定のみの画面は Step2 の主導線外。必要なら最終週以降。）

## 5. 技術制約・開発工数の反映（ここで難所を落とす）

| 項目 | 内容 |
|------|------|
| 標準スタックとの関係 | [`STACK_CHOICE.md`](../STACK_CHOICE.md) に従い、**Next.js + Vercel + Supabase（Auth + Postgres）** を正とする。逸脱する場合はメンターと附録扱いで合意。 |
| データ | Step1 G3 と整合: **DB で材料・叩き・企業行を永続化**。教材レーンでは **Supabase Auth とセット**になりやすいが、必達の言い方は「**ログインまたはDB**」のどちらかで満たす（本テーマは **DB＋本人のみアクセス** が主）。 |
| 緑パターン | [`IDEATION_AND_SCOPE_LADDER.md`](../IDEATION_AND_SCOPE_LADDER.md): **P1（自分用メモ・テキストの保存）＋材料→叩きの1処理**に近い。**画面は一覧／詳細の2枚**で収める。 |
| **除外したもの**（難易度・工数で落とした機能） | Googleドキュメント**自動連携**；**締切・選考ステップのフル管理**；**公式サイトからの設問スクレイピング・完全一致**；**リアルタイム共同編集**；**本格 RBAC（複数ロール）**；**ドキュメントの更新を常時検知するリアルタイム同期分析** |
| 除外理由（1行ずつ） | **深い外部連携**は黄寄りで主導線が分岐する／**選考管理**はP5相当で別の核になりうる／**公式設問の自動取得**はサイト差分・規約で保守不能に近い／**リアルタイム**は黄の典型／**複数ロール**は権限設計が重い／**常時同期**は外部I/Oと監視が重い |

### 追加する機能（このストーリーに「含める場合」の定義）

| 追加 | 内容 |
|------|------|
| **企業名に応じた想定ES設問の更新** | ユーザーが**企業名を入力・確定**したタイミングで、**LLM が「よくある設問」のリストを推測生成**し、**設問欄を動的に並べ替える**（JSON 等で保持してよい）。**実際のエントリーフォームと一致しない場合がある**旨を画面に必ず表示する。 |
| **（任意・別途）志望動機の3パターン** | 同一の生成アクションで**志望動機案を3つ**出す、などは**プロンプトと表示枠の追加**に収め、主導線は **1本**のままにする。 |

### やらないこと（追加機能とあわせて明文化）

| やらない | 理由（1行） |
|----------|-------------|
| **採用サイトの自動スクレイピング**で公式設問を取得する | 仕様差・ログイン壁・利用規約・メンテで **黄〜赤寄り**になりやすい。 |
| **Googleドキュメント／Drive とリアルタイム同期**して常に分析する | **深い外部連携＋リアルタイム**で黄の典型。 |
| **公式フォームと100%一致**を保証する | 本プロダクトは**叩き・参考**であり、**推測ベース**であることを前提にする。 |
| **設問の網羅・全業界テンプレの完全対応** | 仕様が無限に広がり、2週間の核を削る。 |

## 6. スコープ段（A〜D）と推奨

| 案 | 概要 | 画面数 | 1週間リスク | 推奨 |
|----|------|--------|-------------|------|
| **A 最小** | 企業1件・単一画面寄りで「貼る→生成→保存」だけ先に通す。一覧が弱い。 | 1〜2 | G4 の「2画面説明」と Step1 の「企業が増える」に**ずれやすい** | □ |
| **B 標準** | **S1 一覧（追加）＋S2 ワークスペース**。Supabase で `companies` 行をユーザーに紐づけ。 | **2** | LLM API・環境変数・エラー表示のバランス（許容） | ☑ |
| **C ひと伸び** | B に **一覧内検索**や**材料スニペット1ブロック**などを1つだけ。 | 2 | 検索・断片管理で**核（保存・一覧）が遅れる** | □ |
| **D 参考（非推奨）** | GDoc連携・選考ダッシュボード・複数フォーマット完全対応。 | 4+ | **黄〜赤寄り**、2週間の主導線から外れる | □ |

**採用**: **（B 標準）** — Step1 の主導線（企業の追加・一覧から再開・材料→叩き→保存）と一致し、**G4 を2画面で説明しやすい**ため。

## 7. 必達（[`MVP_CHECKLIST.md`](../MVP_CHECKLIST.md)）への対応（見立て）

| 必達 | このストーリーで最終週に満たし方（仮） |

## 9. 難易度が高いときの進め方（メンター相談・代替案のテンプレ）

次のように整理してから **メンター（班・OH）に相談**すると、スコープがブレにくいです。

1. **事実**: 「〇〇（例: 公式から設問を自動取得、GDocリアルタイム同期、志望動機3案＋動的設問＋別機能…）を **この2週間で全部やるのは難易度が高い**」と自分で言語化する。  
2. **必須かの確認**: 「**絶対にやらないといけない**と感じる部分があれば、**一人で抱えずメンターに相談**する（緑帯・必達とのトレードオフを一緒に見る）。」  
3. **代替案**: 「**代替案**は △△（例: **企業名＋LLMで想定設問のみ**／**材料貼付は手動**／**志望動機3案は叩き生成に統合**）なので、**まずはこの形で主導線1本を卒業ラインに届け、その後に伸ばす**」と一文で書く。

**コピペ用の例（そのまま使ってよい）**

> 〇〇はこの2週間では難易度が高いです。**どうしても必須の場合はメンターに相談**します。代替としては「△△」で進めますので、**まずはそこで必達を満たし、公式取得やリアルタイム同期はスコープ外**とさせてください。
