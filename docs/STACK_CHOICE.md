# 推奨技術スタック（標準レーン）

初学者〜40 名規模では選択肢を増やすほどサポートが破綻するため、**標準は 1 系統**に固定する。逸脱は「上級者レーン」として**附録**にまとめ、メンターは標準レーンを優先して答える。

## 開発の主役（AI・編集体験）

| 役割 | 推奨 | 備考 |
|------|------|------|
| IDE | **Cursor** | コード編集・AI とのペアプロが中心。学生向け Pro／教育オファーは**開講前に公式で確認**（[`POLICY_SCALE_AND_COSTS.md`](POLICY_SCALE_AND_COSTS.md)） |
| LLM | **Claude**（Cursor 経由または公式） | 教材・スクリーンショットは Cursor 上の流れを正とする |
| 補助（v2以降で検討） | **Manus** 等 | **v1 では受講生に使わせない**。主役は Cursor のみ |

メンター向けの一言: 「**デバッグとデプロイの質問は Cursor＋リポジトリ上**。**v1 は Manus なし**。」

## 標準（推奨・メイン教材）— アプリとインフラ

| 層 | 選定 | 理由 |
|----|------|------|
| フロント／アプリ | **Next.js**（App Router） | ルーティング・API Route・デプロイ資料が揃いやすい |
| ホスティング | **Vercel** | Next との一体運用、HTTPS 既定 |
| 認証・DB | **Supabase**（Auth + Postgres） | ログインと DB を同一プロバイダで完結しやすい |
| リポジトリ | **GitHub** | 無料枠・CI・学生に馴染みやすい |

**MVP の満たし方（このコースの必達との対応）**

- 「**ログイン**」→ Supabase Auth（メールマジックリンク等で十分）
- 「**DB**」→ Supabase のテーブルに 1 つでも読み書きがあれば可（複雑な正規化は不要）

## 附録 A — 代替レーン（経験者・事情がある人向け）

| 層 | 代替案 | 注意 |
|----|--------|------|
| フロント | **Vite + React** | ルーティング・ビルドを自分で把握する必要 |
| ホスティング | **Netlify** / **Cloudflare Pages** | 設定ファイルが別になる |
| BaaS | **Firebase**（Auth + Firestore） | データモデルが NoSQL |
| 静的のみ | **GitHub Pages** | **ログイン／DB 必達**を満たすには別途 BaaS が必要 |

## 附録 B — やらない方がよい（2 週間では非推奨）

- モバイルネイティブのみ（ストア申請はコース射程外）
- 自前 VPS でのフル管理（セキュリティ・保守の説明コストが高い）
- マイクロサービス分割・複数 DB

## メンター向け一言（インフラ）

「**Next + Vercel + Supabase** 以外は“自己責任レーン”。デプロイと秘密情報で詰まったら標準レーンへの縮小を提案する。」
