# 主催の進め方（本番コホート想定：最大40名・メンターがボトルネックのとき）

**前提（あなたの回答）**: ゴールは **本番に近い開催**。いまいちばん重いのは **メンター人数と確保方法**。

下は **並列で進められるもの** と **順番を崩すと痛いもの** に分けた流れです。

---

## フェーズ0 — メンター設計（最優先）

先に数字と約束を置く。ここが曖昧だと日程・募集人数を決めにくい。**候補はいるが未経験が多い**場合は、研修のスケジュールとティア分けを [`MENTOR_ONBOARDING.md`](MENTOR_ONBOARDING.md) で先に固定する。

| やること | 出力 |
|----------|------|
| **コア常駐の目安** | プランどおり **2〜4名**を「名前またはロール」で確保可否を聞く |
| **火木18〜21のコミット** | 各週 **何人いれば回るか**（班長頼み／チケット制前提なら表にする） |
| **ボランティアの扱い** | 「当日だけ」「月◯時間まで」など**期待値の文章化** |
| **メンター用の読み物** | [`mentor_ops_handbook.md`](mentor_ops_handbook.md)・[`STACK_CHOICE.md`](STACK_CHOICE.md)・[`IDEATION_AND_SCOPE_LADDER.md`](IDEATION_AND_SCOPE_LADDER.md) を一通り渡す日 |
| **未経験メンターの育成** | [`MENTOR_ONBOARDING.md`](MENTOR_ONBOARDING.md) に沿った **T-4週〜** の研修カレンダーと **T1 到達チェック** |

**判断**: コアが **2名未満**で固まらない場合は、先に **募集上限を30名に下げる**／**OHを週1に減らす**／**有償TA** のどれかを検討（開催日だけ先に決めない方が安全）。
