# B2B 向けパッケージ（企業・スポンサー説明用）

無料学生向け開催と両立しつつ、**協賛・次期応用編・採用接点**を検討する企業向けに、そのままコピー＆差し替えできる骨子。

---

## 1. ペルソナと母集団（例・要カスタマイズ）

| 項目 | 内容（案） |
|------|----------------|
| 対象 | 大学生（全学年可の場合は明記） |
| 前提スキル | プログラミング未経験〜初級。**コンソール未経験でも参加可** |
| 規模 | 1 期あたり最大 **40 名**（運営・会場と整合） |
| 形式 | **オフライン／オンライン併用**、同期スタート・卒業式あり |
| 動機 | AI エージェントでものを作りたい／業務改善・BPO・セキュリティ等への関心 |

---

## 2. 到達定義（卒業ライン）

2 週間終了時点で、受講生は次を満たすことを目標とする（詳細は [MVP_CHECKLIST.md](./MVP_CHECKLIST.md)）。

- **Web アプリを HTTPS の公開 URL で提示できる**
- **核機能が 1 本通る**
- **簡易ログインまたは DB 永続化**のいずれかを含む
- **README** と **秘密情報の非公開**を理解している

技術の標準レーンは [STACK_CHOICE.md](./STACK_CHOICE.md)（例: Next.js + Vercel + Supabase）。

---

## 3. 運営体制（開示用）

| 役割 | 内容（案） |
|------|------------|
| コアメンター | **2〜4 名**（社会人ボランティア等） |
| オフィスアワー | **火・木**（初回コホートは **19:00〜21:30** 目安。期ごとに [`FIRST_COHORT_SCHEDULE.md`](FIRST_COHORT_SCHEDULE.md) で確定） |
| サポート方針 | **チケット制**・班の相互サポート。**24 時間以内の即時解決は保証しない** |
| コンプライアンス | ハラスメント防止・個人情報の取り扱いは別紙ポリシーに準拠（要作成） |

## 4. 利用ツール・実費（要確認）

| 項目 | 検討ポイント |
|------|----------------|

## 5. スポンサー権利の型（例）

無料提供と両立するため、**権利は段階化**することを推奨。**協賛ランクは 3 段階**を標準とする（インタビュー反映）。**ロゴの最大幅・掲出位置は契約書に px または % で明記**（[`LP_AND_SPONSOR_RULES.md`](LP_AND_SPONSOR_RULES.md)）。

| レベル | 例（契約で確定） |
|--------|------------------|
| **ランク A（上位）** | LP／資料への **ロゴ（契約数値）**、**開講・卒業の周知**、**5 分スピーチ**枠（任意） |
| **ランク B（中位）** | **ロゴ（小サイズ・フッター等）**、**開講の周知** |
| **ランク C（名称・クレジット）** | **ロゴなし**・**テキストクレジットのみ**、または **コンテンツ協力のみ** |

**避ける**: 受講生の個人情報の無制限提供、必須の採用応募。
