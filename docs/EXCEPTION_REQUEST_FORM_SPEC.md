# 有料インフラ「例外申請」Google フォーム（設計メモ）

**目的**: 原則 **無料枠のみ**（[`POLICY_SCALE_AND_COSTS.md`](POLICY_SCALE_AND_COSTS.md)）から外れる申請を扱う。**濫用を減らす**ため、**自由記述は最小限**にし、**必須項目は多め・記入負荷は高めでよい**（インタビュー反映）。

---

## 運用

| 項目 | 方針 |
|------|------|
| 申請先 | **Google フォーム**（**URL は主催が発行**） |
| 承認 | **社員** |
| **SLA** | **受付から3営業日以内**に返信（承認・却下・追加質問） |

---

## フォーム設計の考え方

- **必須にするもの**: 判断に必要な項目は**すべて必須**（抜けを残さない）。  
- **自由記述**: **あまり書いてほしくない**方針のため、**長文テキストは極力なし**または**字数上限を短く**。  
- **記入負荷**: **チェックボックス・選択式・数値**を多くし、**手間をかけてもよい**（真剣な申請だけ通るイメージ）。  
- 例: 利用サービス名、見積URL／月額、**標準レーンで代替不可な理由**（選択式＋短文）、リポジトリURL、チケット番号 等 — **実装時に列挙して必須化**。
