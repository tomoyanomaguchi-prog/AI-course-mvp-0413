# Discord チャンネル・表示名・命名規則

**目的**: **実名性**と**質問の集約**を両立し、迷子を減らす（[`COMMUNITY_AND_COMMS.md`](COMMUNITY_AND_COMMS.md)）。

---

## 1. チャンネル（最低構成）

| チャンネル | 用途 |
|------------|------|
| `#質問`（または `#support`） | **チケット・技術質問の正（1 本）**（[`mentor_ops_handbook.md`](mentor_ops_handbook.md)） |
| `#自己紹介` | 入ったタイミングで **一言**（大学・やりたいこと等） |
| `#雑談` | 技術以外・軽い連絡（**質問は #質問 へ誘導**） |

**3つに分ける理由（一言ずつ）**: **#質問**＝技術の正と検索性。**#自己紹介**＝顔と文脈。**#雑談**＝水場で #質問 を汚さない。詳細は [`mentor_ops_handbook.md`](mentor_ops_handbook.md) の「3 チャンネルの意図」。

※ 運営・お知らせ用の `#announcements` 等は任意で追加。

---

## 2. 表示名（実名性）

| 項目 | 方針 |
|------|------|
| **受講生・メンター** | **実名を推奨**（インタビュー反映）。**Discord のサーバーニックネーム**で **`姓 名`（全角スペース）** など **読みやすい形式**にそろえる |
| **運営** | 役割がわかる表記（例: `運営 〇〇`）でもよい |

---

## 3. 命名規則（たたき台）

| ルール | 内容 |
|--------|------|
| **基本形** | **`姓 名`**（例: `山田 太郎`）。**半角英数字のみのハンドル**を表示名の正にしない |
| **重複時** | **`姓 名（大学の短称）`** 例: `山田 太郎（〇〇大）` |
| **禁止・非推奨** | **誹謗中傷・他者になりすまし**。**先頭のみ絵文字**は非推奨（検索しづらい） |
| **メンター** | 上記に加え、**ロール**で「メンター」が付く前提なので、表示名に **メンター** と書かなくてよい |

開講前に **#自己紹介** に **サンプル1行**をピン留めするとそろいやすい。
