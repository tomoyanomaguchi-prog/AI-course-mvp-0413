# アイデアソン（課題の掘り下げ）× スコープの三段ライン

**目的**: 「型で縛りすぎない」ことと「迷子にならない」ことの両立。  
流れは **①課題を徹底的に言語化（アイデアソン）→ ②パターン／レベルに当てはめて調整**。順番を逆にしない（先にパターンだけ選ぶと思考が浅くなりやすい）。

---

## ① アイデアソンで掘る（最低限の問い）

ペアまたは班でよい。**誰の・何の・なぜ今**まで掘る。メモはそのまま Day1 の [`DAY1_MVP_TEMPLATE.md`](DAY1_MVP_TEMPLATE.md) に転記する。

| レイヤ | 問い（例） |
|--------|------------|
| 誰 | 具体的な一人（ペルソナ1行でよい）。「みんな」は禁止に近い。 |
| 状況 | いつ・どこで・何が起きている？ |
| 課題 | 何がつらい？ **行動**に落とせる？（例: 「探すのに毎回10分」） |
| 既存の代替 | いまは何で凌いでいる？（スプシ、LINE、紙…） |
| 最小の成功 | 2週間後、「これができたら一人は喜ぶ」は何？ |

ここまでやってから、下の **② ライン** を見せて「自分はどの帯で戦うか」を選ぶ。

---

## ② スコープの三段ライン（全部見せる）

同じ「課題の深さ」でも、**実装の重さ**で難易度が変わる。受講生は **どの帯を狙うか** を宣言し、メンターは Day1 で **帯のズレ**だけ直す。

### 緑：このパターンなら「いける」ライン（標準サポート内）

次の **どれかに当てはまれば**、標準スタック＋メンター OH で到達しやすい想定。

| ID | パターン（例） | データのイメージ |
|----|----------------|------------------|
| P1 | 自分用メモ・タスク・習慣の1画面＋保存 | ユーザーに紐づく行を CRUD |
| P2 | 小さな「投稿／コメント」掲示板 | 1 メインテーブル＋ユーザーID |
| P3 | イベント・申込・参加者一覧 | フォーム送信 → 一覧表示 |
| P4 | 簡易アンケート集計（内部向け） | 回答の保存と集計ビュー |
| P5 | 課題の「提出物URL・期限」を並べるダッシュボード | 少数フィールドの CRUD |

**共通**: 画面はだいたい **一覧／詳細 or フォーム／設定** の3枚以内。外部 API 連携は「読むだけ」かゼロが無難。

---

### 黄：「ここまでいくときつい」ライン（やればOKだが覚悟がいる）

やってはいいが、**メンターは優先度を下げる**／**自分で調査が大半**になる帯。

- リアルタイム双方向（チャット、同時編集、通知の常時配信）
- 複数ロール（管理者・一般・モデレータ）の **権限設計が本格**
- 外部サービスと **OAuth 以外の深い連携**（決済、在庫、地図の高度利用）
- モバイル最適化・PWA・オフラインを「製品レベル」で追う
- **セキュリティが本体**になる題材（脆弱性診断ツール等）

→ どうしてもやるなら **機能を削って緑に落とす**か、**レベルを宣言してスコープを切る**こと。

---

### 赤：「流石に多すぎ」ライン（このコースではアウト）

次は **2週間・この卒業定義では扱わない**。アイデアとしては残してよいが、**別プロジェクト**扱い。

- 金銭・個人情報・医療・法律判断に触れる本番運用
- アプリストア申請・ネイティブアプリが主役
- チーム開発前提の巨大設計（マイクロサービス、複数リポジトリ）
- **秘密情報・本番データ**を扱う実データ接続（[`MVP_CHECKLIST.md`](MVP_CHECKLIST.md) の禁止と重なる）

---

## ③ 「型」と「やってみたい」のバランスの取り方（運営ルール）

| やること | 理由 |
|----------|------|
| **①を先に**やらせる | パターン先選びは「誰の課題か」が薄くなりがち |
| **緑のパターンは「例」**として見せる | 該当しなくても、**同等の重さ**なら自作テーマでよい |
| **黄に入る要素があれば1つだけ** | 複数あるとほぼ赤に近づく |
| Day1 で **「狙う帯：緑／黄（削定後）」** を書かせる | メンターがズレを早期修正 |

---

## ④ 受講生向け一言（スライドにそのまま使える）

- 課題の掘り下げは **深くてよい**。  
- 実装の重さは **この表で帯を選ぶ**。  
- **深い課題 × 重い実装** はこの期間では両立しにくいので、**まず緑で動くもの**を出し、残りは「次の2週間」のネタにする。

---

## ⑤ メンター用：会話のフック

- 「面白いね。それ、**データは誰の行を1枚**に収まる？」→ 収まらなければ緑に分解できないか聞く。  
- 「それ、**黄のどれに当たる？** 一個なら一緒に削る」  
- 赤に近い → 「アイデアは残して、**デモではダミーデータだけ**にしよう」
