# MVP 必達チェックリスト（2週間コース）

受講生の最終提出が「卒業ライン」を満たすための**必須条件**と、**やってはいけないこと**を定義する。メンター・最終審査はこの表を基準にする。

## 必達（すべて Yes）

| # | 項目 | 合格の目安 |
|---|------|------------|
| 1 | **公開 URL** | HTTPS で第三者がブラウザからアクセスできる（ローカルのみ不可） |
| 2 | **核機能 1 本道** | 課題の主導線が通る（例: 入力 → 処理 → 結果表示／登録） |
| 3 | **永続化（どちらか必須）** | **簡易ログイン**（メール／OAuth 等）**または** **DB／永続ストレージ**のいずれか |
| 4 | **レスポンシブ最低限** | 主要画面がスマホ幅で破綻しない（完璧なデザインは不要） |
| 5 | **README** | 起動手順、環境変数の置き場、既知の制限が書いてある |
| 6 | **秘密情報の扱い** | API キー等がリポジトリに直書きされていない（`.env` / ホスト側シークレット） |

## 推奨（できれば満たす）

| # | 項目 | 目安 |
|---|------|------|
| R1 | エラーハンドリング | 想定外入力でもアプリが落ちない／ユーザーに分かる表示 |
| R2 | 友達フィードバックの記録 | 誰に見せて、何を直したかが一言で追える |
| R3 | ライセンス | 利用ライブラリの表記（README または画面フッター） |

## 禁止・失格になりうる行為（秘密情報・法令・倫理）

以下は**指導で明示的に禁止**し、発覚時は運営判断で提出取り下げ・参加停止の対象になりうる。

1. **認証情報の漏えい**
   - リポジトリ・スクリーンショット・チャットログへの **API キー・DB 接続文字列・秘密鍵** の貼付
   - 公開リポジトリに `.env` をコミットすること
2. **他人の個人情報・企業秘密の無断利用**
   - 実在の第三者の許可なく個人情報を扱う機能に使うこと
   - 勤務先・クライアントの機密データを教材・**利用中の AI ツール**に投入すること
3. **悪用・迷惑行為につながる機能**
   - スパム送信、なりすまし、無断スクレイピングの自動化など
4. **サービス利用規約違反**
   - Cursor・LLM・ホスティング・API 提供者の ToS に反する利用

## メンター・受講生向けショート版

- **公開できる？** URL を教えて動く？  
- **ログイン or DB** どちらか入っている？  
- **鍵は漏れてない？** `.env` と README を確認した？  
- **スマホで見れる？**
